## Описание
Сейчас при инициализации приложения активный экран всегда сбрасывается на `feed`, даже если пользователь до этого был в разделе `rules` или `profile`.
Из-за этого после обновления страницы теряется контекст.

Нужно сохранять выбранную вкладку нижнего меню (`feed | rules | profile`) и восстанавливать её при следующем открытии страницы.

## Current behavior
1. Пользователь открывает вкладку «Правила».
2. Обновляет страницу.
3. Приложение открывает «Лента».

## Expected behavior
- После reload/open восстанавливается последняя активная вкладка.
- Если сохранено невалидное значение — fallback на `feed`.

## Acceptance Criteria
- [ ] Активная вкладка сохраняется при переключении.
- [ ] При старте читается сохранённое значение и открывается соответствующий экран.
- [ ] Для невалидного/пустого значения используется `feed`.
- [ ] Не ломается текущее переключение вкладок кликом.

## Tech notes
- Можно использовать `localStorage` (например, `bazardrive_active_tab`) или URL hash.
- Логику удобно встроить в `setActiveScreen` + блок инициализации.
