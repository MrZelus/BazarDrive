## 1. Общая структура экрана (сверху вниз)

Экран включает следующие секции в порядке отображения:

1. Шапка профиля
2. Быстрые статусы
3. Навигация по вкладкам
4. Основной контент активной вкладки
5. Нижняя панель действий

## 3. Секция 1 — Шапка профиля

**Расположение:** верхняя часть экрана.

**Состав:**
- фото пользователя;
- ФИО;
- роль пользователя;
- форма деятельности;
- налоговый режим;
- общий статус аккаунта.

**Пример содержимого:**
- Фото
- Алексей Смирнов
- Водитель такси
- ИП
- УСН «Доходы»
- Статус: Активен

**Назначение:**
Обеспечить мгновенную идентификацию пользователя и текущего состояния его рабочего профиля.

## 4. Секция 2 — Быстрые статусы

**Расположение:** сразу под шапкой.

**Формат:** горизонтальные карточки/компактные блоки.

**Карточки:**
- Документы: 8/8 подтверждены
- Разрешение такси: активно
- Выплаты: подключены
- ОСАГО: действует до 18.09.2026

**Назначение:**
Дать пользователю обзор критически важных статусов в одном месте.

## 5. Секция 3 — Навигация по вкладкам

**Расположение:** под блоком быстрых статусов.

**Вкладки:**
- Обзор
- Такси / ИП
- Документы
- Выплаты
- Безопасность

**Примечание:** эталонная структура профиля — 5 вкладок в указанном порядке; README и UI должны соответствовать этому списку.

**Поведение:**
- активная вкладка визуально выделяется;
- переключение выполняется без перезагрузки страницы;
- контент ниже обновляется в зависимости от выбранной вкладки.

## 6. Секция 4 — Основной контент активной вкладки

Основная рабочая область экрана. Содержимое зависит от выбранной вкладки.

### 5.1. Вкладка «Обзор»

**Блоки:**
- рейтинг;
- город;
- зона работы;
- автомобиль;
- количество поездок;
- доход за день/неделю.

**Пример:**
- Рейтинг: 4.9
- Город: Казань
- Зона работы: центр / аэропорт
- Авто: Toyota Prius 50
- Поездок выполнено: 1284
- Доход за неделю: 32 400 ₽

### 5.2. Вкладка «Такси / ИП»

**Блоки:**
- ИНН;
- ОГРНИП;
- дата регистрации ИП;
- налоговый режим;
- разрешение такси;
- регион работы;
- данные автомобиля;
- водительское удостоверение.

**Пример:**
- ИНН: ************
- ОГРНИП: ***************
- Налоговый режим: УСН «Доходы» 6%
- Разрешение такси: активно
- Регион: Татарстан
- Права: действуют до 2029

### 5.3. Вкладка «Документы»

**Формат:** список документов с иконками и статусами.

**Список:**
- паспорт;
- ИНН;
- ОГРНИП;
- разрешение на такси;
- водительское удостоверение;
- СТС;
- ОСАГО;
- диагностическая карта.

**Для каждого документа отображаются:**
- название;
- дата загрузки;
- срок действия;
- статус.

### 5.4. Вкладка «Выплаты»

**Блоки:**
- реквизиты;
- расчётный счёт;
- история выплат;
- доход за период;
- налоговый резерв;
- выгрузка отчёта.

**Пример:**
- Банк: Т-Банк
- Выплаты: подключены
- Доход за месяц: 124 000 ₽
- Резерв на налог 6%: 7 440 ₽

### 5.5. Вкладка «Безопасность»

**Блоки:**
- смена пароля;
- подтверждение телефона;
- подтверждение email;
- 2FA;
- активные устройства;
- история входов.

## 7. Секция 5 — Нижняя панель действий

**Расположение:** внизу экрана или в виде закреплённой панели.

**Кнопки:**
- Редактировать профиль
- Загрузить документ
- Открыть выплаты
- Настройки безопасности

**Логика адаптации:**
- на вкладке «Документы» приоритет — загрузка документов;
- на вкладке «Выплаты» — реквизиты и отчёты;
- на вкладке «Безопасность» — защита аккаунта.

## 9. События интерфейса

| Событие                         | Что происходит                          |
| ------------------------------- | --------------------------------------- |
| Нажатие на вкладку              | Переключается активный раздел профиля   |
| Нажатие «Редактировать профиль» | Открывается форма редактирования        |
| Нажатие «Загрузить документ»    | Открывается загрузчик файла             |
| Нажатие на документ             | Открывается просмотр карточки документа |
| Нажатие «Заменить документ»     | Загружается новая версия файла          |
| Нажатие «Удалить документ»      | Открывается подтверждение удаления      |
| Нажатие «Открыть выплаты»       | Переход к финансовому разделу           |
| Переключение 2FA                | Меняется статус двухфакторной защиты    |
| Ошибка API                      | Показывается error-state или banner     |

## 11. Рекомендации по адаптиву

### Мобильный экран

- быстрые статусы лучше показывать в 2 строки карточек;
- вкладки можно сделать горизонтально скроллируемыми;
- нижнюю панель действий закрепить снизу.

### Десктоп

- быстрые статусы в одну линию;
- вкладки под шапкой;
- действия можно вынести вправо или вниз контентного блока.
