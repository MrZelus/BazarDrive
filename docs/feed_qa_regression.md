# QA-набор по ленте: smoke + регрессия

Документ фиксирует минимальный набор ручных и автотестовых проверок для PR-цикла по ленте.

## 1) Smoke (быстрый прогон)

> Цель: за 10–15 минут убедиться, что базовый сценарий публикации работает.

### SMK-01: Публикация валидного поста
- Предусловие: есть валидный профиль (name + email или phone).
- Шаги:
  1. Открыть вкладку `Лента`.
  2. Ввести валидный текст поста.
  3. Нажать «Опубликовать».
- Ожидаемо:
  - `POST /api/feed/posts` возвращает `201`.
  - Новый пост отображается в списке.

### SMK-02: Переход в профиль при невалидном профиле
- Предусловие: профиль без обязательных полей.
- Шаги:
  1. На вкладке `Лента` заполнить черновик.
  2. Нажать «Опубликовать».
- Ожидаемо:
  - Черновик сохраняется.
  - UI переключается в `Профиль` → `Документы`.
  - После сохранения профиля происходит авто-возврат в `Лента` и повтор публикации.

### SMK-03: Поведение при rate limit
- Шаги:
  1. Быстро отправить несколько постов подряд.
- Ожидаемо:
  - API отвечает `429` и возвращает `retry_after` / `Retry-After`.
  - UI показывает время ожидания и не теряет текст черновика.

## 2) Регрессия (обязательный набор)

### API-контракт
- [ ] Все endpoint'ы из `docs/openapi.yaml` доступны и соответствуют фактическим статусам/полям.
- [ ] Ошибки `400/404/409/429` сериализуются по зафиксированному контракту.
- [ ] `X-Request-ID` возвращается в API-ответах JSON.

### UX и навигация
- [ ] Нижнее меню корректно переключает `Лента ↔ Профиль ↔ Правила`.
- [ ] Карта переходов и сценарии соответствуют `docs/feed_navigation_publish_flow.md`.
- [ ] Автовозврат к публикации после сохранения профиля работает без перезагрузки страницы.

### Правила публикации и модерация
- [ ] Запрещённые темы блокируются с понятной ошибкой.
- [ ] Спам/избыточные ссылки блокируются.
- [ ] Локальный precheck отображает предупреждения до отправки.
- [ ] Сквозной сценарий `publish -> moderation -> approved` подтверждён:
  1. `POST /api/feed/posts` создаёт запись в статусе модерации.
  2. Модератор одобряет запись.
  3. `GET /api/feed/approved` возвращает одобренный пост.

### Взаимодействия в ленте (реакции, комментарии, поиск)
- [ ] Реакции сохраняются серверно (`POST/DELETE /api/feed/posts/{id}/reactions`) и не теряются после перезагрузки.
- [ ] `my_reaction` и aggregate-счётчики (`reactions.like/dislike`) совпадают с payload backend после установки/смены/снятия реакции.
- [ ] Комментарии создаются, редактируются и удаляются только владельцем (`POST/PATCH/DELETE /api/feed/posts/{id}/comments`).
- [ ] При `403/404/400` для комментариев и реакций UI показывает понятные сообщения (без «тихих» сбоев).
- [ ] Поиск и фильтры в ленте возвращают стабильный и повторяемый список результатов.
- [ ] Infinite scroll / pagination не дублируют и не пропускают элементы при последовательной подгрузке.

### Документы водителя
- [ ] `POST /api/driver/documents` создаёт документ.
- [ ] `PATCH /api/driver/documents/{id}` обновляет документ.
- [ ] `DELETE /api/driver/documents/{id}` удаляет документ.
- [ ] Дубликаты корректно возвращают `409 duplicate_document`.

## 3) Автотесты (локальный минимум)

```bash
python -m unittest tests.test_feed_api_smoke_publish -v
python -m unittest tests.test_feed_publish_profile_navigation_flow -v
python -m unittest tests.test_driver_documents_contract_sync -v
```

## 4) Release-gate

Фича ленты считается готовой к релизу, если:
1. Smoke-прогон успешен.
2. Все пункты регрессии закрыты.
3. Автотесты из блока выше проходят на CI без flaky-падений.
