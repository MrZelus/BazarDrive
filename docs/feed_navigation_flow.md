# Лента: навигационный флоу между разделами

Документ фиксирует актуальный navigation-flow для пользовательского сценария **Лента → Профиль → Одобренное** с учётом контрактов API.

## 1) Карта навигации (основной сценарий)

```text
[Лента / draft]
   |  POST /api/feed/posts
   |  (профиль невалиден)
   v
[Профиль / документы]
   |  POST /api/feed/profiles
   |  (профиль валиден)
   v
[Лента / повтор публикации]
   |  GET /api/feed/posts
   v
[Список публикаций]
   |  GET /api/feed/approved
   v
[Одобренная лента]
```

## 2) Узлы API в порядке прохождения

1. `POST /api/feed/posts` — создать публикацию из черновика.
2. `POST /api/feed/profiles` — сохранить профиль, если публикация была остановлена валидацией.
3. `GET /api/feed/posts` — перечитать персональную ленту после успешной отправки.
4. `GET /api/feed/approved` — получить единый список одобренных постов и объявлений.

## 3) Правила навигации UI

- При невалидном профиле черновик публикации сохраняется в localStorage.
- Пользователь переводится в `Профиль` на вкладку `Документы`.
- После валидного сохранения профиля происходит автоматический возврат в `Ленту` и повтор отправки.
- Секция `Одобрено` отображает только записи со статусом `approved` (серверная агрегация).

## 4) Контрольные точки для QA

- Переходы между вкладками: `Лента ↔ Профиль ↔ Правила`.
- Восстановление черновика после forced redirect в профиль.
- Повтор публикации после `POST /api/feed/profiles` без ручной перезагрузки страницы.
- Актуальность одобренной ленты после публикации и модерации (`GET /api/feed/approved`).
