# BazarDrive Permissions Matrix

Матрица прав для основных ролей системы:
- `guest`
- `passenger`
- `driver`
- `moderator`
- `admin`

## Обозначения
- ✅ разрешено
- ❌ запрещено
- ⚠️ зависит от политики, контекста или требует явного ограничения в RBAC

## Рекомендуемая интерпретация ролей

### guest
- читает правила и ленту
- может создать publication profile
- может публиковать контент в guest feed при наличии валидного профиля публикации
- не может создавать taxi orders и не может управлять driver compliance

### passenger
- оформляет заказы такси
- смотрит свои заявки и связанный пользовательский профиль
- не имеет доступа к driver compliance и moderation-функциям

### driver
- управляет driver profile, compliance, документами и сменой
- открывает/закрывает путевой лист
- выходит online и принимает заказы только при прохождении guard/compliance checks
- видит summary, reminders и score

### moderator
- модерирует публикации
- проверяет publication profiles
- проверяет документы водителя и принимает review decisions
- не должен менять системные роли пользователей без отдельного admin-доступа

### admin
- имеет полный доступ
- управляет ролями и статусами пользователей
- может выполнять override-операции в moderation и compliance

## Security notes

1. Все действия уровня `moderator` и `admin` должны проходить через явную RBAC-проверку.
2. Для `driver` доступ к операциям `go_online`, `accept_order`, `open_shift` должен дополнительно зависеть от compliance state и active waybill state.
3. Для `guest` и `passenger` полезно отделять:
   - базовый аккаунт (`users`)
   - профиль публикации (`guest_profiles`)
   - доменные сущности заказов/поездок.
4. `moderator` лучше ограничить review-функциями без права менять глобальные роли и статусы учетных записей.
5. `admin` должен иметь audit trail для чувствительных операций: смена роли, смена статуса, override moderation/compliance decisions.
