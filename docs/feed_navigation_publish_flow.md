# Лента/Профиль/Правила: карта переходов и публикация

Документ описывает единый UX/API флоу для разделов **Лента**, **Профиль**, **Правила** и сценария публикации.

## 1) Карта переходов между разделами

### Основные переходы (нижнее меню)
- `Лента` → `Профиль` (редактирование профиля, документы, статус готовности к публикации).
- `Лента` → `Правила` (правила платформы, публикации, шаблоны документов).
- `Профиль` → `Лента` (возврат к публикации и просмотру постов).
- `Правила` → `Лента`/`Профиль` (быстрое возвращение к публикации или заполнению профиля).

### Принудительный переход при публикации
- Если пользователь пытается опубликовать пост без валидного гостевого профиля:
  1. остаётся черновик текста поста;
  2. выполняется редирект в `Профиль` (вкладка `Документы`);
  3. показывается подсказка о необходимых полях (имя + email/телефон).

### Продолжение после сохранения профиля
- После успешного `POST /api/feed/profiles` при наличии сохранённого черновика:
  1. пользователь автоматически возвращается в `Лента`;
  2. публикация автоматически повторяется с тем же текстом;
  3. при успехе черновик очищается.

## 2) API-узлы флоу

- `POST /api/feed/profiles` — сохранить/обновить профиль гостя.
- `POST /api/feed/posts` — создать публикацию.
- `GET /api/feed/posts` — обновить список ленты после публикации.

## 3) Тест-кейсы

### Happy path
1. Пользователь с валидным профилем вводит текст на вкладке `Лента`.
2. Нажимает «Опубликовать».
3. API возвращает `201 Created`.
4. Пост появляется в ленте.

### Validation
1. Пользователь без валидного профиля вводит текст и нажимает «Опубликовать».
2. Происходит переход в `Профиль` → `Документы`.
3. Пользователь заполняет профиль и сохраняет.
4. Приложение автоматически возвращает пользователя в `Лента` и повторяет публикацию.

### Rate limit (`429 Too Many Requests`)
1. Пользователь публикует несколько постов слишком быстро.
2. API возвращает `429` + `retry_after`/`Retry-After`.
3. UI показывает понятное сообщение с ожиданием в секундах.
4. Текст черновика не теряется, можно повторить позже.

### Навигация между разделами
1. Пользователь перемещается через нижнее меню: `Лента ↔ Правила ↔ Профиль`.
2. Активная вкладка сохраняется между перезагрузками (через localStorage).
3. При возврате в `Профиль` виден актуальный статус готовности профиля к публикации.
