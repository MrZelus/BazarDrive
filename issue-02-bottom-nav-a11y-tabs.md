## Описание
Нижнее меню визуально работает как табы, но сейчас реализовано как обычные кнопки без полного ARIA tabs-паттерна.
Это ухудшает доступность для пользователей screen reader и keyboard-only.

Нужно сделать семантику и поведение, соответствующие tabs-паттерну:
- контейнер: `role="tablist"`;
- кнопки: `role="tab"`, `aria-controls`, `aria-selected`;
- панели: связь с табами;
- навигация клавишами (Left/Right/Home/End).

## Current behavior
- Есть клик по кнопкам и переключение экранов.
- Нет полноценной семантики tabs.
- Нет навигации стрелками.

## Expected behavior
- Табы корректно озвучиваются assistive-технологиями.
- Фокус и выбор вкладки управляются с клавиатуры по общему паттерну tabs.
- Состояния активной/неакивной вкладки корректно отражены в ARIA.

## Acceptance Criteria
- [ ] Контейнер нижнего меню имеет `role="tablist"`.
- [ ] Каждая кнопка имеет `role="tab"` и `aria-controls`.
- [ ] Активная вкладка отмечается через `aria-selected="true"` (остальные `false`).
- [ ] Поддержаны Left/Right/Home/End для перемещения по табам.
- [ ] Скрытые панели не фокусируются и корректно помечены для AT.
- [ ] Регрессии кликового поведения отсутствуют.
